# Matriz de Evaluación de Riesgos – Enfoque de Control Preventivo

## 🔷 Propósito

Evaluar el nivel de exposición al riesgo de un proceso organizacional, considerando la capacidad del sistema para anticipar, prevenir y gestionar eventos antes de su materialización.

---

## 🔷 Estructura de evaluación

| Elemento                     | Descripción                                      | Nivel Bajo | Nivel Medio | Nivel Alto |
|----------------------------|--------------------------------------------------|-----------|------------|-----------|
| Identificación de riesgos   | Reconocimiento previo de eventos potenciales     |           |            |           |
| Prevención estructural      | Existencia de controles antes del proceso        |           |            |           |
| Calidad de la información   | Disponibilidad y confiabilidad de datos          |           |            |           |
| Oportunidad de la información | Información disponible a tiempo para decidir   |           |            |           |
| Integración del control     | Nivel de integración en la toma de decisiones    |           |            |           |
| Control en tiempo real      | Seguimiento simultáneo al proceso                |           |            |           |
| Responsabilidad distribuida | Participación de actores en el control           |           |            |           |

---

## 🔷 Interpretación

- Mayor concentración en niveles altos → Riesgo elevado  
- Predominio de niveles medios → Riesgo moderado  
- Predominio de niveles bajos → Control preventivo sólido  

---

## 🔷 Uso práctico

Esta matriz permite:

- Evaluar procesos críticos  
- Identificar debilidades estructurales  
- Rediseñar sistemas de control  
- Apoyar la toma de decisiones  

---

## 🔷 Enfoque diferencial

A diferencia de matrices tradicionales:

- No se limita a probabilidad e impacto  
- Integra control, información y decisión  
- Evalúa la capacidad preventiva del sistema  

---

## 🔷 Conclusión

El análisis de riesgos no debe centrarse únicamente en el evento.

Debe centrarse en la capacidad del sistema para evitar que ocurra.

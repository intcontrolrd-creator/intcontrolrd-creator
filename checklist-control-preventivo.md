# Checklist de Control Preventivo

## 🔷 Propósito

Este checklist permite evaluar si un proceso organizacional está diseñado bajo un enfoque de control preventivo, orientado a la anticipación de riesgos.

---

## 🔷 Instrucciones de uso

Marque cada criterio según corresponda:

- [ ] Cumple  
- [ ] Parcialmente cumple  
- [ ] No cumple  

---

## 🔷 1. Diseño del proceso

- [ ] El proceso está claramente definido  
- [ ] Existen responsables identificados  
- [ ] Las actividades están estructuradas y documentadas  
- [ ] Se han identificado puntos críticos  

---

## 🔷 2. Prevención de riesgos

- [ ] Se han identificado riesgos antes de la ejecución  
- [ ] Existen controles previos definidos  
- [ ] El sistema evita errores antes de que ocurran  
- [ ] No depende exclusivamente de controles posteriores  

---

## 🔷 3. Información

- [ ] La información está disponible antes de la decisión  
- [ ] Los datos son completos y confiables  
- [ ] Existe integración entre fuentes de información  
- [ ] Se evita la fragmentación de datos  

---

## 🔷 4. Toma de decisiones

- [ ] Las decisiones se toman con base en información validada  
- [ ] El control forma parte del proceso de decisión  
- [ ] No se ejecutan acciones sin validación previa  

---

## 🔷 5. Control en tiempo real

- [ ] El proceso cuenta con mecanismos de seguimiento simultáneo  
- [ ] Se detectan desviaciones durante la ejecución  
- [ ] No se depende únicamente de auditorías posteriores  

---

## 🔷 6. Responsabilidad de control

- [ ] El control no está concentrado en un solo actor  
- [ ] Los responsables del proceso asumen funciones de control  
- [ ] Existe cultura de prevención  

---

## 🔷 Evaluación general

- Nivel de control preventivo:  
  - [ ] Alto  
  - [ ] Medio  
  - [ ] Bajo  

---

## 🔷 Conclusión

Este instrumento permite identificar si un proceso está orientado hacia la anticipación de riesgos o si mantiene un enfoque reactivo tradicional.

El objetivo no es solo evaluar, sino rediseñar el proceso hacia un modelo preventivo.
